# دليل تشغيل نظام "مفقود" للذكاء الاصطناعي 🚀

هذا الدليل يشرح الخطوات اللازمة لتشغيل المشروع على جهازك المحلي أو على سيرفر.

## 📋 المتطلبات المسبقة
قبل البدء، تأكد من تثبيت البرامج التالية:
1.  **Python 3.9+**: لغة البرمجة الأساسية.
2.  **Redis**: يستخدم كوسيط (Broker) لمهام Celery.
3.  **Git**: لتحميل المشروع (اختياري).

---

## 🛠️ خطوات الإعداد (اول مرة فقط)

### 1. إنشاء بيئة وهمية (Virtual Environment)
من داخل مجلد المشروع، قم بتشغيل الأوامر التالية:
```bash
python -m venv venv
```
ثم تفعيلها:
- **Windows:** `.\venv\Scripts\activate`
- **Linux/Mac:** `source venv/bin/activate`

### 2. تثبيت المكتبات المطلوبة
```bash
pip install -r requirements.txt
```

---

## 🚀 تشغيل النظام

لتشغيل النظام بالكامل، تحتاج لفتح **3 واجهات أوامر (Terminals)** مختلفة:

### الواجهة الأولى: تشغيل Redis
تأكد من تشغيل سيرفر Redis أولاً. إذا كنت تستخدم Windows، يمكنك تشغيل `redis-server.exe`.

### الواجهة الثانية: تشغيل سيرفر FastAPI
هذا السيرفر مسؤول عن واجهة الموقع ونقاط اتصال الـ API:
```bash
uvicorn app.main:app --reload
```
يمكنك الآن الدخول للموقع عبر: `http://localhost:8000`

### الواجهة الثالثة: تشغيل عمال المهام (Celery Worker)
هذا الجزء مسؤول عن معالجة الصور والفيديوهات في الخلفية:
```bash
celery -A app.celery_app worker --loglevel=info -P solo
```
> [!TIP]
> في بيئة الإنتاج (Linux)، استخدم `--autoscale=10,3` بدلاً من `-P solo` لتحسين الأداء.

---

## 🐳 التشغيل باستخدام Docker (الخيار الأسهل)
إذا كنت تملك Docker، يمكنك تشغيل النظام بالكامل (الـ API، العمال، و Redis) بضغطة واحدة:

```bash
docker-compose up --build
```

**مميزات تشغيل Docker:**
1. يقوم تلقائياً بتثبيت كل المكتبات والاعتمادات (بما فيها OpenCV).
2. يشغل Redis تلقائياً داخل حاوية (Container) خاصة.
3. يربط الخدمات ببعضها البعض دون تعقيدات الـ Path والـ Windows.

- **توثيق الـ API (Swagger):** `http://localhost:8000/docs`
